# SG Analytics Engine — Журнал архитектурных решений

> Каждое важное решение фиксируется здесь. Это предотвращает повторное обсуждение и сохраняет контекст между сессиями.

---

## 2026-05-04 — Выбор AI модели для аналитического агента

**Решение:** Kimi K2.6 через Gonka AI (основная) + Claude API (Vision + финальный текст)

**Причина:**
- Kimi K2.6 в 4 раза дешевле Claude Sonnet при сопоставимом качестве для кода
- Tool invocation success rate 96% — достаточно для итерационного цикла
- Галлюцинации на данных устранены архитектурно через E2B
- Claude API остаётся для Vision (Kimi слабый, #26 из 115) и финального текста

**Альтернативы отклонены:**
- CoPaw LoRA на Fireworks — требует DevOps, GPU деплой, разработчик
- Только Claude API — дороже в 4 раза при масштабе
- DeepSeek — серверы в Китае, риск для US/CA рынка

---

## 2026-05-04 — Execution engine

**Решение:** E2B sandbox

**Причина:**
- Изолированное выполнение Python — безопасность для мультипользователей
- $100 стартовых кредитов бесплатно
- ~$0.001 за запрос при масштабе
- Устраняет галлюцинации: модель не называет цифры — вычисляет кодом

**Альтернативы отклонены:**
- Локальный Python — небезопасно для мультипользователей
- Cloudflare Workers Python — ограниченная среда выполнения

---

## 2026-05-04 — Оркестратор

**Решение:** Cloudflare Durable Objects для длинных запросов

**Причина:**
- Обычный Worker лимит 30 сек — 10 итераций × 3-5 сек = 30-50 сек, граничное значение
- Durable Objects нет лимита времени + stateful (хранит контекст итераций)
- Уже в инфраструктуре Cloudflare — не новый сервис

**Альтернативы отклонены:**
- Modal.com — нужен Python разработчик для деплоя
- FastAPI сервер — нужна отдельная инфраструктура

---

## 2026-05-04 — Стратегия запуска

**Решение:** Сначала универсальный CSV Analyst, потом специализация

**Причина:**
- Нет реальных пользователей — нет данных для выбора специализации
- 3 недели до рабочего MVP vs 8 недель до специализированного продукта
- Данные от бета-пользователей покажут куда специализироваться
- $25-35 до бета vs $235-345 в старом плане

**Альтернативы отклонены:**
- Сразу SG Competitive Intelligence — рынок переполнен без дифференциации
- Сразу MarketLens — не валидирован спрос

---

## 2026-05-04 — Данные через Kimi (риск Китай)

**Решение:** Использовать DeepInfra как провайдер Kimi K2.6 (серверы в США)

**Причина:**
- Kimi K2.6 открытые веса (MIT лицензия) — можно запускать на любой инфраструктуре
- DeepInfra хостит K2.6 на серверах в США
- Данные пользователей физически остаются в США
- Решает compliance для US/CA рынка

**Fallback:** Claude API если DeepInfra нестабилен — замена одной строки в конфиге

## 2026-05-05 — Временное переключение на Claude API для отладки

**Решение:** Временно использовать Claude API вместо Kimi K2.6 (DeepInfra) для прохождения end-to-end теста

**Причина:** Нулевой баланс на DeepInfra аккаунте. Пополнить баланс на deepinfra.com (~$10).

**ВАЖНО — ВЕРНУТЬ ПОСЛЕ ОТЛАДКИ:**
- Пополнить баланс на https://deepinfra.com → Billing
- В kimi.ts изменить baseUrl обратно на DeepInfra и модель на Kimi K2.6
- Проверить что DEEPINFRA_API_KEY актуален в Cloudflare secrets

**Альтернативы отклонены:**
- Остаться на Claude API навсегда — в 20 раз дороже при масштабе
