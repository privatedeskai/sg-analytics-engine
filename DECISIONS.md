## 2026-05-04 — Выбор AI модели для аналитического агента

**Решение:** Kimi K2.6 через DeepInfra (основная) + Claude API (Vision + финальный текст)

**Причина:**
- Kimi K2.6 в 4 раза дешевле Claude Sonnet при сопоставимом качестве для кода
- Tool invocation success rate 96% — достаточно для итерационного цикла
- Claude API остаётся для Vision и финального текста

**Альтернативы отклонены:**
- Только Claude API — дороже в 4 раза при масштабе
- DeepSeek — серверы в Китае, риск для US/CA рынка

## 2026-05-06 — Режим работы Kimi K2.5/K2.6: Instant vs Thinking

**Решение:** Использовать Instant mode для итерационного агента

**Контекст:**
Kimi K2.5 и K2.6 — гибридные модели с двумя режимами работы:

| Режим | Когда использовать | Температура | max_tokens | Скорость |
|---|---|---|---|---|
| **Instant** | Агентские циклы, короткие задачи, написание кода | 0.6 | 1500-3000 | ~3-5s |
| **Thinking** | Сложный reasoning, математика, глубокий анализ | 1.0 | ≥16000 | ~20-60s |

**Причина выбора Instant:**
- Диагностика показала: при max_tokens=1500 и Thinking mode модель тратит все токены на reasoning_content и не оставляет места для реального ответа (content_chars=0, reasoning_chunks=1401)
- Для написания Python скрипта на каждой итерации Thinking избыточен
- Instant mode: быстро, дёшево, предсказуемо

**Как включить Instant mode на DeepInfra:**
```json
{
  "thinking": {"type": "disabled"},
  "temperature": 0.6
}
```

**Как включить Thinking mode (если понадобится в будущем):**
```json
{
  "temperature": 1.0,
  "max_tokens": 16000
}
```

**Где смотреть в коде:** `worker/src/kimi.ts` — метод `callAPIStreaming`, параметры тела запроса.

**Когда может понадобиться Thinking mode:**
- Финальный summary после всех итераций — более глубокий текст
- Сложные аналитические вопросы где нужен многошаговый reasoning
- Э2+ когда добавим специализированные продукты

## 2026-05-06 — Архитектура итерационного агента (Вариант 2)

**Решение:** Резюме между итерациями + ранний выход по сигналу модели

**Суть:**
- Между итерациями передаются короткие summary (~30 слов), а не сырой вывод
- Kimi сама сигнализирует `enough: true` когда данных достаточно для ответа
- Это обеспечивает сходимость и предотвращает раздувание контекста

**Файлы:** `worker/src/kimi.ts`, `worker/src/orchestrator.ts`
