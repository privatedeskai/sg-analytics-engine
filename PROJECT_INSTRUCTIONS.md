## 1. О ПРОЕКТЕ

**Название:** SG Analytics Engine
**Тип:** Universal AI Analytics Platform — веб-SaaS
**Цель:** Универсальный AI-аналитик. Пользователь загружает данные или указывает источник — система автономно анализирует, находит паттерны, даёт рекомендации.
**Рынок:** США / Канада

### Контекст
Проект является развитием ScreenGuide for Shopify. Вся инфраструктура уже создана и работает:
- Cloudflare Workers + KV
- Vercel (веб-приложение)
- Claude API
- GitHub репозиторий
- Рабочий процесс: Claude пишет код → Олег деплоит

## 3. АРХИТЕКТУРА ДВИЖКА

Источник данных (CSV / API / BrowserUse)
        ↓
Нормализация → стандартный табличный формат
        ↓
Оркестратор (Cloudflare Durable Objects)
        ↓
Kimi K2.6 → пишет Python
        ↓
Judge0 CE → выполняет → результат + графики
        ↓
Kimi K2.6 → видит результат → следующая итерация (до 10)
        ↓
Claude API → Vision (если нужно) + финальный текст
        ↓
Output: текст + Chart.js + PNG matplotlib

## 12. ИЗВЕСТНЫЕ ОГРАНИЧЕНИЯ СТЕКА

- **E2B SDK** несовместим с Cloudflare Workers (ESM/gRPC проблема) — не использовать
- **Piston API** закрыт с 15 февраля 2026 — не использовать
- **Judge0 CE** (`ce.judge0.com`) — текущий execution engine, без ключа, pandas/numpy включены
- **Claude API модель** — использовать `claude-sonnet-4-5` (не `claude-sonnet-4-20250514`)
- **KV binding** в wrangler.toml называется `KV` (не `ANALYTICS_KV`)
- **Durable Object класс** называется `AnalysisOrchestrator` (не `Orchestrator`)

## 13. РЕГЛАМЕНТ ВЫБОРА ВНЕШНИХ РЕШЕНИЙ (ОБЯЗАТЕЛЬНЫЙ)

Применяется при выборе любого: API сервиса, npm/pip пакета, облачной платформы, SDK, инструмента деплоя, библиотеки, готового решения.

**Алгоритм (выполнять каждый раз без исключений):**

1. **Сформулировать критерии** — что нужно: функция, совместимость со стеком, надёжность

2. **Найти минимум 2-3 варианта** — не останавливаться на первом знакомом, поискать через web_search

3. **Отсортировать по стоимости трений** — от простого к сложному:
   - ✅ Без регистрации / без ключа / без карты → **приоритет 1**
   - ✅ Бесплатный ключ, без карты → **приоритет 2**
   - ⚠️ Бесплатный план, нужна карта → **приоритет 3, предупредить Олега**
   - ❌ Платный → только если нет бесплатного варианта

4. **Показать сравнение** перед рекомендацией — таблица или список с плюсами/минусами

5. **Обосновать выбор** — почему выбранный вариант лучше остальных

**Пример правильного решения (Judge0 vs RapidAPI):**
- RapidAPI/Judge0: бесплатно, но требует карту → приоритет 3
- ce.judge0.com: бесплатно, без регистрации, без карты → приоритет 1 ✅
- Выбор: ce.judge0.com
