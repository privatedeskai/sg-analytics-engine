## 1. О ПРОЕКТЕ

**Название:** SG Analytics Engine
**Тип:** Universal AI Analytics Platform — веб-SaaS
**Цель:** Универсальный AI-аналитик. Пользователь загружает данные или указывает источник — система автономно анализирует, находит паттерны, даёт рекомендации.
**Рынок:** США / Канада

### Контекст
Проект является развитием ScreenGuide for Shopify. Вся инфраструктура уже создана и работает:
- Cloudflare Workers + KV
- Vercel (веб-приложение)
- Claude API
- GitHub репозиторий
- Рабочий процесс: Claude пишет код → Олег деплоит

## 3. АРХИТЕКТУРА ДВИЖКА

Источник данных (CSV / API / BrowserUse)
        ↓
Нормализация → стандартный табличный формат
        ↓
Оркестратор (Cloudflare Durable Objects)
        ↓
Kimi K2.6 → пишет Python
        ↓
E2B Sandbox → выполняет → результат + графики
        ↓
Kimi K2.6 → видит результат → следующая итерация (до 10)
        ↓
Claude API → Vision (если нужно) + финальный текст
        ↓
Output: текст + Chart.js + PNG matplotlib

## 5. РЕГЛАМЕНТ РАБОТЫ (ЖЁСТКИЙ — ВСЕГДА СОБЛЮДАТЬ)

### 5.1 Формат ответов
- **ПРАВИЛО 0:** Все шаги ОДНИМ сообщением — никогда не разбивать
- **ПРАВИЛО 1:** Всё для копирования — только в блоках кода
- **ПРАВИЛО 2:** PowerShell команды в отдельных блоках, без &&, группировать через ;
- **ПРАВИЛО 3:** Полные файлы целиком — никогда частичные вставки
- **ПРАВИЛО 4:** Claude не задаёт вопросы Олегу об архитектуре — решает сам
- **ПРАВИЛО 5:** ВСЕ URL — ТОЛЬКО В БЛОКАХ КОДА. Никогда не писать URL inline в тексте. Это касается любых адресов: GitHub, Vercel, Cloudflare, Workers, внешние сервисы — абсолютно все.
- **ПРАВИЛО 5 (повтор 1):** URL inline = нарушение регламента. Каждый URL — отдельный блок кода.
- **ПРАВИЛО 5 (повтор 2):** Перед отправкой сообщения — проверить: есть ли URL вне блока кода? Если да — переместить в блок.
- **ПРАВИЛО 5 (повтор 3):** Кликабельные ссылки в тексте ЗАПРЕЩЕНЫ. Только блок кода.
- **ПРАВИЛО 5 (повтор 4):** Примеры запрещённого формата — "Открыть https://github.com/..." или "[ссылка](https://...)" — оба варианта нарушают регламент.
- **ПРАВИЛО 5 (повтор 5):** Правильный формат для любого URL: отдельный блок кода без исключений.
- **ПРАВИЛО 5 (повтор 6):** Это правило применяется в каждом сообщении, в каждой сессии, без исключений и напоминаний со стороны Олега.
- **ПРАВИЛО 5 (повтор 7):** Если Claude написал URL вне блока кода — это ошибка того же уровня что и частичная вставка файла.
- **ПРАВИЛО 5 (повтор 8):** Самопроверка перед отправкой: найти все символы "http" в сообщении → убедиться что каждый из них внутри блока кода.

## 11. РЕШЕНИЯ ПО ИНТЕРФЕЙСУ (зафиксировано)

### Выбранный вариант: Гибридный — Чат + Дашборд

**Э1:** левая колонка — чат (~40%), правая — живой дашборд (~60%)
**Э2+:** боковая навигация + чат + дашборд

### Ключевые принципы UI
- Пользователь не выбирает режим — пишет в чате на естественном языке
- Дашборд обновляется по ходу анализа (не после) — итерация за итерацией
- Прогресс итераций — содержательные статусы, не спиннер

### Шаблоны запросов
Хранение в Cloudflare KV: вопрос + Python скрипт + схема данных + расписание
Два режима запуска:
- Быстрый (~5 сек): скрипт без AI если структура данных не изменилась
- Полный (~60 сек): регенерация через Kimi K2.6 если данные изменились
- Система выбирает режим автоматически

### История отчётов
KV ключи:
- templates:{user_id}:{template_id}
- reports:{user_id}:{date}
- schedule:{template_id}
